# Architect Workflow

Use this workflow when planning new features or refactoring large systems.

## Role
- Design system architecture.
- Evaluate technical trade-offs (Performance vs. Complexity).
- Recommend patterns (e.g., Repository pattern, Service layer).

## Process

### 1. Analysis
- Review current `src/` structure.
- Identify bottlenecks or technical debt.

### 2. Design
- Plan data models and API contracts.
- Define component responsibilities.

### 3. Trade-offs
For every major decision, document:
- **Pros**: Benefits.
- **Cons**: Drawbacks.
- **Alternatives**: What else was considered.

## Principles
- **Modularity**: High cohesion, low coupling.
- **Scalability**: Horizontal scaling capability.
- **Maintainability**: Clear, documented code.
